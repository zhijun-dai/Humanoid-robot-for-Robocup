# 场地数据核验与落地方案

## 1. 结论（先看这个）

1. `el05.stp` 是 CAD 几何文件，但不是完整比赛场地模型。
2. 现在仓库已有可用的完整场地图纸：`dxf_out/2025人形竞技全能场地图纸.dxf`。
3. 该 DXF 可直接提取场地外框、赛道内外边界、刻度文本等几何信息，已可作为仿真建模主数据源。

## 2. 核验证据

### 2.1 `el05.stp` 的单位与尺寸

- 文件里多处长度单位定义为毫米（`SI_UNIT(.MILLI.,.METRE.)`）。
- 对 `CARTESIAN_POINT` 全量统计后，外包范围约为：
  - X: `-28.8047 .. 41.8020`，跨度 `70.607`
  - Y: `-31.0077 .. 46.8010`，跨度 `77.809`
  - Z: `-18.5 .. 80.25`，跨度 `98.75`

这与比赛场地量级（米级，约 3660mm x 2440mm）不一致，说明它更像局部零件/组件，而不是整场地。

### 2.2 仓库中可用但不完整的场地信息

- 任务文本有流程信息：`Task.md`、`CVpart/README.md`。
- 有候选位图：`前人代码/前人摄像头1/a.bmp`，分辨率仅 `56x44`，不足以作为正式场地图。

### 2.3 新增：DXF 图纸自动提取结果

源文件：`dxf_out/2025人形竞技全能场地图纸.dxf`

关键头信息：

1. `ACADVER = AC1024`（AutoCAD 2010 代际）
2. `INSUNITS = 4`（毫米）

从 `ENTITIES` 中提取到的关键几何：

1. 外框矩形四条边坐标：
  - X 范围：`1772.877810431` 到 `5432.877810430999`
  - Y 范围：`2419.412157625529` 到 `4859.412157625528`
  - 尺寸：`3660 x 2440 mm`
2. 赛道内外边界：两条闭合 `LWPOLYLINE`，含圆弧 bulge 参数。
3. 赛道刻度文字：`0, 5, 10, ... , 60`（TEXT/MTEXT 可读）。

说明：当前已能稳定提取到场地外框和赛道主轮廓，起跑线/障碍/二维码位仍需结合规则页最终确认。

## 3. 当前可用的场地参数基线（截图 + DXF 交叉验证）

1. 场地外框：`3660mm x 2440mm`
2. 赛道总长：`6140mm`
3. 赛道宽度：`350mm`
4. 限宽门最窄：`240mm`

注：障碍、二维码、起跑线的精确坐标与尺寸，仍需从图纸图层语义或规则正文补齐。

## 4. 立刻可执行的落地步骤

### 步骤 A：建立统一参数文件（今天就能完成）

先在项目内约定唯一数据源（建议 JSON/YAML），字段至少包含：

- field_width_mm
- field_height_mm
- track_length_mm
- track_width_mm
- narrow_gate_min_mm
- start_line_pose
- qr_slots
- obstacle_pose

### 步骤 B：生成仿真可用底图（先跑视觉）

1. 先按上述参数画高分辨率黑白底图（建议 5 px/cm 到 10 px/cm）。
2. 导入仿真地面纹理，先验证视觉识别链路。
3. 之后再补精确特征（限宽门、木条、起跑线、二维码位）。

### 步骤 C：做一次尺寸一致性检查

检查项：

1. 仿真地面物理尺寸是否为 `3.66m x 2.44m`
2. 赛道宽是否为 `0.35m`
3. 起跑线、限宽门位置是否与规则图一致

## 5. 你现在只需要补充的信息

1. 起跑线的最终定义（图层名或规则文字对应位置）。
2. 木条障碍和二维码位的精确坐标/摆放规则。

补齐后可直接产出：

1. 一份可落地的参数文件（含坐标）
2. 一张可直接导入仿真的标准场地纹理图
3. 一份最小验收清单（确保尺寸和视觉一致）
