# 📝 大创申报书创新点包装指南

> 先快速科普：**大创 = 大学生创新创业训练计划**，分三类：
> - 🔬 创新训练（科研导向，发论文/专利）
> - 💼 创业训练（商业导向，做产品/公司）
> - 🚀 创业实践（落地导向，需企业导师）
> 
> 你们适合申报 **"创新训练项目"**，核心是：**有明确科学问题 + 可行技术路线 + 预期成果可考核**。

---

## 一、项目定位建议（一句话包装）

### ❌ 不要这样写（太实诚）：
> "做一个小型人形机器人参加2025中国机器人大赛，用强化学习控制行走。"

### ✅ 建议这样写（有格局）：
> **"面向低成本平台的人形机器人轻量化强化学习方法研究与系统实现"**  
> 或  
> **"基于仿真迁移的小型人形机器人自适应步态控制关键技术研究"**

**核心逻辑**：把"参加比赛"包装成"验证方法"，把"复现开源"包装成"技术创新"。

## 二、创新点提炼（按申报书常用套路）

### 🔹 创新点1：低成本约束下的轻量化RL架构（技术型）

**问题**：现有仿人机器人强化学习研究多依赖高性能算力平台（如Jetson AGX, ¥10,000+），难以在教育/竞赛场景普及。

**我们的方案**：
- 提出 **~~~控制架构**("Jetson Nano + STM32"双层控制架构),比如：
  - 上层（Jetson）：轻量化策略网络（<1MB，INT8量化），负责高层决策
  - 下层（STM32）：实时FOC电机控制（200Hz），负责底层执行
- 设计 **状态/动作空间压缩方法**：从30维观测→12维关键特征，推理延迟<20ms

**创新价值**：
> 首次将端到端强化学习策略部署到¥1,200级边缘算力平台，为低成本人形机器人智能控制提供新范式。

**可考核成果**：
- ✅ 开源轻量化策略网络代码（GitHub）
- ✅ 在¥7,000总成本平台上实现稳定行走（视频+数据）
- （✅ 发表1篇会议论文（如IEEE CASE/ROBIO））

---

### 🔹 创新点2：面向小样本场景的域随机化策略（方法型）

**问题**：传统Sim-to-Real需大量真实数据微调，但学生团队难以获取足量真机数据。

**我们的方案**：
- 提出 **"任务引导的域随机化"（Task-Guided Domain Randomization）**：
  - 不是随机所有参数，而是**聚焦关键不确定性**：电机扭矩系数、地面摩擦、传感器噪声
  - 结合比赛任务（行走+越障+动作），设计**多任务奖励函数**，提升策略泛化性
- 开发 **快速仿真-真机迭代流程**：1次真机测试→反馈修正仿真参数→重新训练，闭环优化

**创新价值**：
> 针对学生团队"数据少、时间紧"的痛点，提出小样本友好的迁移学习框架，降低强化学习落地门槛。

**可考核成果**：
- ✅ 开源域随机化配置工具（Python包）
- ✅ 真机首次行走成功率≥70%（对比基线+30%）
- ✅ 形成《低成本人形机器人Sim-to-Real实践指南》（技术报告）

---

### 🔹 创新点3：模块化开源人形机器人平台（工程型）

**问题**：现有开源人形机器人要么成本过高（如Unitree H1），要么文档不全，学生难以复现。

**我们的方案**：
- 基于 **"参考设计+自主改进"** 模式：
  - 机械：参考Mini π/ Qmini URDF，针对EDULITE 05电机重新设计安装结构
  - 电气：自主设计STM32+Jetson通信板（可拿规则8.4附加分）
  - 软件：封装"感知-决策-控制"标准接口，支持传统PID/RL策略热切换
- 提供 **从零到一的实施文档**：采购清单、装配视频、调试手册、故障排查

**创新价值**：
> 打造首个面向中国大学生竞赛的"千元级"开源人形机器人平台，推动智能机器人教育普惠化。

**可考核成果**：
- ✅ 开源完整项目仓库（BOM+CAD+URDF+代码+文档）
- ✅ 支持3种以上电机型号快速适配（扩展性验证）
- ✅ 被≥2支其他高校队伍复用（影响力证明）

## 三、申报书核心模块写法建议

### 📋 1. 立项依据（为什么做？）

```
【研究背景】
• 人形机器人是具身智能重要载体，但高成本制约教育普及
• 强化学习在仿真中表现优异，但真机部署面临"最后一公里"难题
• 中国机器人大赛等赛事为技术验证提供场景，但缺乏低成本解决方案

【国内外现状】
• 国际：Boston Dynamics/Unitree性能强但成本高（¥10万+）
• 国内：高校研究多聚焦算法，缺少"算法+硬件+系统"全栈实践
• 缺口：¥10,000内、可复现、支持智能算法的小型人形平台

【本项目定位】
• 不追求"最强性能"，而追求"最优性价比"
• 以竞赛任务为驱动，验证轻量化RL方法的工程可行性
• 产出开源平台，降低后续团队入门门槛
```

### 🎯 2. 研究内容（做什么？）

```
【总体目标】
构建总成本≤¥7,000的小型人形机器人系统，实现：
① 平整场地稳定行走（速度≥0.2m/s）
② 二维码识别+指定动作执行（准确率100%）
③ 10mm障碍自主跨越（成功率≥90%）

【具体任务】
任务1：轻量化强化学习策略设计
  - 状态空间压缩：从30维→12维关键特征
  - 网络结构优化：3层MLP+INT8量化，模型<1MB
  - 奖励函数设计：平衡+前进+节能+任务完成多目标加权

任务2：高效Sim-to-Real迁移方法
  - 关键参数辨识：电机/摩擦/噪声的实测标定流程
  - 任务引导域随机化：聚焦不确定性，减少训练开销
  - 真机快速迭代：1次测试→参数修正→重训练的闭环

任务3：模块化系统集成与开源
  - 硬件：EDULITE 05电机适配结构+STM32/Jetson通信板
  - 软件：ROS2节点封装+策略热切换接口
  - 文档：从零装配指南+故障排查手册+教学案例
```

### 📊 3. 技术路线（怎么做？）

```
【阶段1：仿真验证（3-5月）】
  PyBullet建模 → PPO训练 → 域随机化 → 策略导出

【阶段2：真机部署（6-8月）】
  硬件装配 → 底层驱动 → 策略量化 → 首次行走调试

【阶段3：任务集成（9-10月）】
  视觉联调 → 状态机设计 → 稳定性测试 → 比赛验证

【阶段4：成果固化（11-12月）】
  代码开源 → 文档撰写 → 论文/专利投稿 → 大创结题
```

## 四、避坑指南（评审专家可能质疑的点）

### ❓ 质疑1："不就是复现开源项目吗？创新在哪？"

**应对话术**：
> "我们不是简单复现，而是在**低成本约束下做工程创新**：  
> - 开源项目多用高端电机（¥2000+/个），我们适配¥300的EDULITE 05  
> - 开源项目多用高性能算力板，我们压缩模型到Jetson Nano可运行  
> - 开源项目缺少中文实施文档，我们提供从零到一的国产化指南  
> **创新不在'从无到有'，而在'从贵到普惠'。**"

### ❓ 质疑2："强化学习真能比PID稳吗？别到时候比赛倒一地"

**应对话术**：
> "我们采用**双保险策略**：  
> - 默认使用传统PID控制（确保完赛底线）  
> - RL策略作为'增强模式'，在仿真验证充分后启用  
> - 设计安全监控模块，异常时自动切换回PID  
> **科研探索不冒进，比赛成绩有保障。**"

### ❓ 质疑3："7000元预算真的够吗？"

**应对话术**：
> "我们已做详细成本核算（见附件）：  
> - 电机12×¥300=¥3,600（已谈妥学生价）  
> - 主控+传感器+结构=¥3,400（3D打印用学校资源）  
> - 预留¥500备用金（应对器件损坏）  
> **且大创经费可覆盖部分硬件，实际团队自筹压力<¥3,000。**"
