## 视觉任务
### 一、比赛核心任务总览
机器人从起跑线出发 → 逆时针双足行走614cm环形赛道 → 识别3个二维码并执行动作 → 跨越红色木条障碍 → 到达终点

评分公式：
- 总分 = 行进竞速得分(0-60) + 二维码得分(20×3) + 越障得分(60) - 扣分(出界/倒地-5分/次) + 附加分(0-20)

## 赛场调参版 checklist
按以下顺序执行：光照 → 阈值 → ROI → 串口联调。不要跳步。

### 1. 光照检查（先做）
- [ ] 打开 OpenMV IDE 的 Frame Buffer，确认画面无明显过曝/欠曝。
- [ ] 机器人静止放在赛道中线，摄像头保持“向前、略向下”的固定姿态。
- [ ] 观察赛道边缘在画面中的位置是否稳定（至少观察 10 秒）。
- [ ] 若画面抖动明显，先处理安装固定和供电问题，再进入下一步。

### 2. 阈值调参
- [ ] 先只开一个功能调参（先巡线或先二维码，避免互相干扰）。
- [ ] 巡线：调整灰度阈值与像素门限，目标是边缘线可稳定检出。
- [ ] 二维码：在 6 张二维码样本上测试，确认 1-6 都能被识别。
- [ ] 每次改参数后，至少连续测试 20 帧，再决定是否保留。
- [ ] 把最终可用参数记录到代码注释或文档，避免现场遗忘。

### 3. ROI 调整
- [ ] 在当前机位下，优先把 ROI 放在画面下半区（更接近地面边缘）。
- [ ] 左右 ROI 分别覆盖左右边缘，不要压到画面中央。
- [ ] 调整 ROI 后，观察输出方向码是否更稳定（直走/左转/右转）。
- [ ] 若误判频繁，先缩小 ROI 再微调阈值。

### 4. 串口联调（最后做）
- [ ] 核对串口参数：串口号、波特率、校验位与主控完全一致。
- [ ] 核对协议：二维码动作码与巡线方向码是否和主控定义一致。
- [ ] 在终端同时观察两侧输出：OpenMV 端打印 + 主控端接收日志。
- [ ] 连续测试流程：识别二维码 -> 主控执行动作 -> 恢复巡线。
- [ ] 至少跑 3 轮完整流程，确认无卡死、无重复触发、无漏触发。

### 5. 赛前快速复核（可选）
- [ ] 现场光照变化后，重复步骤 1-2 的快速检查（3 分钟内完成）。
- [ ] 若时间紧，优先保证“二维码识别 + 串口动作触发”稳定。
