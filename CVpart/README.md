## 赛场调参版 checklist
按以下顺序执行：光照 → 阈值 → ROI → 串口联调。不要跳步。

### 1. 光照检查（先做）
- [ ] 打开 OpenMV IDE 的 Frame Buffer，确认画面无明显过曝/欠曝。
- [ ] 机器人静止放在赛道中线，摄像头保持“向前、略向下”的固定姿态。
- [ ] 观察赛道边缘在画面中的位置是否稳定（至少观察 10 秒）。
- [ ] 若画面抖动明显，先处理安装固定和供电问题，再进入下一步。

### 2. 阈值调参
- [ ] 先只开一个功能调参（先巡线或先二维码，避免互相干扰）。
- [ ] 巡线：调整灰度阈值与像素门限，目标是边缘线可稳定检出。
- [ ] 二维码：在 6 张二维码样本上测试，确认 1-6 都能被识别。
- [ ] 每次改参数后，至少连续测试 20 帧，再决定是否保留。
- [ ] 把最终可用参数记录到代码注释或文档，避免现场遗忘。

### 3. ROI 调整
- [ ] 在当前机位下，优先把 ROI 放在画面下半区（更接近地面边缘）。
- [ ] 左右 ROI 分别覆盖左右边缘，不要压到画面中央。
- [ ] 调整 ROI 后，观察输出方向码是否更稳定（直走/左转/右转）。
- [ ] 若误判频繁，先缩小 ROI 再微调阈值。

### 4. 串口联调（最后做）
- [ ] 核对串口参数：串口号、波特率、校验位与主控完全一致。
- [ ] 核对协议：二维码动作码与巡线方向码是否和主控定义一致。
- [ ] 在终端同时观察两侧输出：OpenMV 端打印 + 主控端接收日志。
- [ ] 连续测试流程：识别二维码 -> 主控执行动作 -> 恢复巡线。
- [ ] 至少跑 3 轮完整流程，确认无卡死、无重复触发、无漏触发。

### 5. 赛前快速复核（可选）
- [ ] 现场光照变化后，重复步骤 1-2 的快速检查（3 分钟内完成）。
- [ ] 若时间紧，优先保证“二维码识别 + 串口动作触发”稳定。



## 1
针对摄像头向下俯视赛道所导致的透视畸变问题（画面中直线呈现上窄下宽的梯形，弯道信息被压缩），我们设计了分层ROI与弯道预判相结合的视觉寻迹算法。首先，在不进行完整透视变换的情况下，将图像划分为近、中、远三个动态兴趣区域，并依据其物理意义分配不同的控制权重：近端ROI权重较低，主要抑制抖动；中端ROI权重最高，作为主控依据；远端ROI权重适中，重点用于弯道预判。通过实时计算各ROI内黑线中心的加权平均，得到当前的主控偏移量，同时提取远端ROI的偏移方向与幅度，将其乘以一个自适应系数叠加至转向指令，从而实现“远端先行”的弯道提前响应，有效弥补俯视视角带来的滞后效应。

在控制策略上，我们引入非线性转向映射，将图像偏移量通过S形函数转换为舵机或电机差速角度，使图像边缘的响应趋于平缓，避免过度转向。同时采用双模式PID控制：当远端偏移量较小（直道状态）时使用较小比例系数和适度积分，保证直线行驶的稳定性；当远端偏移量超过阈值（弯道状态）时切换至更高比例系数并限制积分累积，确保快速入弯与平稳出弯。此外，算法还包含动态阈值自适应功能，根据图像灰度直方图实时调整二值化参数，以应对光照变化；并设计了线丢失处理机制，在短暂丢失黑线时按历史转向保守前行，持续丢失则启动原地搜索，提高系统的鲁棒性。

## 2
视觉寻迹采用“灰度采样 + 多点拟合”的传统算法方案。考虑到摄像头向下 30 度安装，地面信息主要集中在图像下半部分，因此设定感兴趣区域（ROI）仅处理下半图，并将分辨率降至 QQVGA（160×120）以提升帧率。通过现场标定黑白灰度阈值，有效隔离光照干扰，确保赛道边界识别的稳定性。
核心检测逻辑采用多条水平扫描线策略。在 ROI 区域内设置 5-7 条横向扫描线，逐行搜索黑白跳变点以确定赛道左右边界。针对直道场景，使用最小二乘法拟合直线边界；针对弯道场景，引入二次曲线拟合评估曲率。同时采用“近处纠偏 + 远处前瞻”的双层策略，利用远处扫描点提前预判弯道趋势，辅助主控调整步态与速度。
最终，算法计算图像中心与赛道中心的横向偏差及曲率信息，经滑动平均滤波平滑后，通过 UART 串口实时下发至主控单元。通信协议采用简洁的帧格式，确保数据传输的低延迟与高可靠性。现场只需 5 分钟即可完成阈值标定，适应不同场馆的光照变化，为机器人高速稳定行走提供视觉决策依据。

## main1.py 思路

### 1) 30度俯视相机模型（从真实安装姿态出发）
- 假设相机相对地面俯视约30度，安装高度约17cm。
- 采用简化针孔模型，把每一行像素 `y` 映射为地面前向距离 `z(cm)`。
- 关键关系：`z = h / tan(ray_angle)`，其中 `ray_angle = pitch + vertical_angle(y)`。
- 同时为每一行计算 `cm_per_px`，把横向像素偏差转成厘米偏差，减少“上远下近”透视带来的尺度不一致。

### 2) 寻迹主线（左右边线中位线）
- 每个ROI内布置多条水平扫描线，逐行找左右边界跳变点。
- 使用左右边界的中位线作为该ROI中心，避免单点噪声影响。
- 在近/中/远三层ROI分别估计：
	- 横向偏差（cm）
	- 航向角误差（deg，基于地面坐标拟合）
	- 置信度（有效行比例 + 线宽稳定性）

### 3) 融合误差与控制
- 融合误差由四部分构成：
	- 中心偏差 `base_err_cm`
	- 远端前瞻偏差 `far_err_cm`
	- 曲率代理 `curve_err = far_err_cm - near_err_cm`
	- 航向角误差 `angle_err`
- 再经过一阶平滑 + 双模式PID + S形非线性映射输出转向，兼顾稳定与响应。

### 4) 什么时候该转弯（距离+偏差联合触发）
- 只看“偏差”容易晚转，需叠加“距离”判断。
- 当前实现采用触发门控 `turn_gate`：
	- 距离因子：越接近转弯区（约42cm到16cm）权重越大。
	- 偏差因子：远端偏差超过阈值（约3.5cm）后增强。
	- 两者相乘得到 `turn_gate`，动态提高前瞻增益并提前进入弯道控制模式。

### 5) 直线与弯道的量化标准（建议）
- 直线稳定：
	- `|base_err_cm| <= 2.0` 且 `|angle_err| <= 4 deg`
- 缓弯：
	- `2.0 < |curve_err| <= 5.0 cm`
- 中急弯：
	- `|curve_err| > 5.0 cm` 或 `turn_gate > 0.35`

### 6) 当前版本可观测输出
- 主循环日志已输出：阈值、转向、横向误差(cm)、角度误差(deg)、远端距离(cm)、转弯门控、模式、置信度、丢线状态与FPS。
- 可直接用于赛场调参与后续“动作层”联动阈值设计。
