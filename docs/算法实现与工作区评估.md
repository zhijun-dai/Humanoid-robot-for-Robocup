# 算法实现与工作区评估

## 一、这次到底更新了什么
前一阶段控制器逐步演化后，存在一个核心偏差：对双边界跑道使用了单线优先跟踪策略。这会导致在多线同时可见时目标线漂移，进而出现“看起来在算偏差，但车实际跑偏”的情况。

本次已改为：
- 主策略：左右边界优先，中心线控制。
- 兜底策略：仅单边可见时，用历史赛道宽度估计中心线。
- 记忆策略：保留历史中心位置和历史赛道宽度，提升连续性。

核心代码位置：
- Webots 控制器主实现：Webots/controllers/line_follow_transfer/line_follow_transfer.py

## 二、当前巡线主逻辑（已实现）
### 1. 图像分层
- 使用近/中/远三层 ROI。
- 每层 ROI 多条水平扫描线采样。

### 2. 每条扫描线如何找中心
- 先根据阈值分出轨道色像素（黑/白可自动判断）。
- 在该行提取所有候选线段 run（长度在 min_line_width ~ max_line_width 之间）。
- 优先在 hint 中心左右各选一条边界，形成左右边界对。
- 若左右边界对宽度在 min_track_width ~ max_track_width 之间，则中心线 = 两边界中点。
- 若只能找到单边，则使用历史赛道宽度 lane_width_hint 推算中心线作为兜底。

### 3. ROI 层结果
每个 ROI 输出：
- center_cm：该层横向偏差（地面厘米）
- dist_cm：该层前向距离
- angle：该层拟合方向角
- conf：该层置信度（有效行比例、线宽稳定性、双边/单边质量综合）
- lane_width_px：该层估计赛道宽度像素

### 4. 控制融合
- near 层为主纠偏。
- far 与 near 的差作为曲率代理，形成弯道前瞻。
- 按 turn_gate 动态切直道/弯道控制。
- 丢线时短时保守，长时定向搜线。
- 速度按 turn_gate 与丢线状态自适应下降。

## 三、如何判断“跑得好不好”
建议统一三层标准：

### A. 在道判定（硬标准）
单帧 in_lane 满足：
- 未丢线
- conf >= in_lane_min_conf
- |ex| <= in_lane_abs_ex_cm

### B. 连续性判定（稳定标准）
- in_lane_ratio（在道率）
- max_out_lane_streak_sec（最长连续离道时长）

### C. 运动性判定（性能标准）
- mean_speed（平均速度）
- low_speed_ratio（长期低速占比）
