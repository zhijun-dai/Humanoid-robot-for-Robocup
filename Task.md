### 2.3 视觉开发技术栈（待验证项）
硬件平台：OpenMV Cam H7 系列（单板一体化，含摄像头+外部SDRAM+外部Flash）
编程语言：MicroPython (嵌入式) / Python (PC预研)
核心库：
  - 二维码：`img.find_qrcodes()`（OpenMV内置）
  - 颜色识别：`img.find_blobs(...)`（HSV阈值）
  - 串口通信：`UART(x, baud)`（以联调结果为准）

开发流程：
  PC端(OpenCV)算法预研 → 阈值/逻辑验证 → MicroPython重写 → OpenMV IDE烧录
说明：阈值、ROI、滤波与防抖参数均为待标定项，不作为最终验收参数。

### 2.4 通信协议设计（视觉→主控，待联调确认）
帧格式：`[帧头][命令字][校验和]`（3字节，简单可靠）

帧头：0xAA (固定)
命令字定义：
  0x01-0x06 → 识别二维码1-6
  0x10      → 检测到红色木条
  0x11      → 木条已通过
  0xFF      → 心跳包(1Hz，确认在线)
校验和：(帧头 + 命令字) & 0xFF

示例：
  识别二维码3 → `0xAA 0x03 0xAD`
  检测到木条   → `0xAA 0x10 0xBA`

联调确认清单：
- 主控是否按该命令字表执行动作
- 串口配置是否与主控一致（串口号、波特率、校验位）
- 是否保留心跳包（0xFF）

### 5.3 竞速得分计算公式
完成全程：得分 = 60 + (tmax - t) / (tmax - tmin) × 40
未完成全程：得分 = 60 × s / 614

其中：
  t = 实际用时，tmax = 4分钟，tmin = 最短用时(动态)
  s = 实际行走路程(cm)，614 = 赛道总长

## 🤖 七、SIM参数盘点（当前目录）

本节用于回答：当前机械与电机相关数据里，哪些已经能用于仿真，哪些仍是刚性缺失。

### 7.1 当前已具备（可支撑基础SIM）

1. 机器人拓扑与关节结构
- 已有连杆层级、父子关系、关节轴、关节类型。
- 可用于导入仿真并驱动关节运动。

2. 连杆质量与惯量
- 已有每个连杆的质量、质心、惯量矩参数。
- 可用于刚体动力学基础计算。

3. 几何与碰撞网格
- 已有各连杆 STL 网格。
- 可用于可视化与基础碰撞近似。

4. 关节限位与基础能力上限
- 已有每个关节的角度上下限、effort、velocity 字段。
- 可用于动作范围约束与基础控制边界。

5. 关节命名清单
- 已有关节名配置，可用于训练代码中的关节索引映射。

说明：以上数据已足够完成“模型导入、关节驱动、站姿与简单动作测试”的基础SIM。

### 7.2 当前刚性缺失（影响训练真实性与Sim2Real）

1. 执行器真实物理参数（高优先级）
- 缺少电机力矩-转速曲线、堵转力矩、空载转速、效率曲线。
- 缺少减速比与传动效率、回程间隙（backlash）定量参数。

2. 关节耗散参数（高优先级）
- 虽有字段，但缺少有效 `damping` 与 `friction` 数值。
- 这会直接影响仿真稳定性与步态真实性。

3. 控制链路参数（高优先级）
- 缺少每关节控制模式对应参数（位置/速度/力矩）与统一控制频率标定值。
- 缺少命令延迟、执行延迟、采样周期抖动等时序参数。
- 缺少可直接用于训练的关节 PD 参数初值（Kp/Kd）配置文件。

4. 接触模型参数（高优先级）
- 缺少脚底-地面静摩擦/动摩擦、恢复系数。
- 缺少接触刚度与阻尼参数。

5. 传感器噪声模型（中高优先级）
- 缺少 IMU/编码器噪声、偏置漂移、量化误差参数。
- 无噪声模型会导致策略过拟合“理想仿真”。

### 7.3 结论（给当前阶段）

1. 现有数据能做：
- 基础SIM验证（模型能否动、动作是否合理、关节是否越限）。

2. 现有数据不能直接保证：
- 高可信强化训练结果和稳定真机迁移（Sim2Real）。

3. 实施建议：
- 先用现有参数完成训练程序与环境打通。
- 再优先补齐“执行器参数 + 接触参数 + 关节摩擦阻尼 + 控制延迟/PD”。
